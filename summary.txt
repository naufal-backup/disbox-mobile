DISBOX & DISBOX-WEB FOLDER CREATION FIX SUMMARY

Fixed the bug where creating a folder inside another folder would place it in the root directory.

Changes made:
1. AppContext.jsx (disbox & disbox-web):
   - Fixed optimistic path generation for new folders by adding a missing slash between the current path and the new folder name.
   - Added an explicit `refresh(true)` call after the folder creation mutation completes to ensure the state is correctly synced from the server.
   - Updated the `createFolder` useCallback dependency array to include `refresh`.

2. LoginPage.jsx (disbox):
   - Fixed JSX syntax error: Added missing closing </div> for methodSelector.
   - Fixed missing Globe import from lucide-react.
   - Verified that both Windows and Linux builds in GitHub Actions are successful.

3. Rubberband Selection Fix:
   - Fixed precision issue when UI Scale is not 100%. Coordinates are now correctly normalized by dividing by uiScale to account for CSS zoom.
   - Lowered rubberband z-index to 150 (below Transfer Panel's 200) so selection doesn't overlap the panel.
   - Applied to both disbox and disbox-web.

4. Version Bump:
   - Bumped disbox version to 4.9.0 in package.json to trigger a new official desktop release.

5. Drag & Drop Upload Overlay Fix:
   - Added missing CSS styling for .dropOverlay in FileGrid.module.css.
   - Enhanced the overlay with a semi-transparent background, blur effect, dashed border, and a bouncing upload icon animation.
   - Localized the "Drop to upload" text in FileGrid.jsx.

6. Context Menu Fix:
   - Fixed issue where the context menu remained open when the delete confirmation popup appeared.
   - Now, the context menu closes immediately when the "Delete" action is selected (both for single and bulk delete), matching the behavior of Move and Copy actions.
   - Applied to both disbox and disbox-web.

7. Disbox Web Title Enhancement:
   - Added favicon (icon.png) to browser tab for disbox-web.
   - Added emoji (📦) to browser title tag in index.html.
   - Added Cloud icon and "Web" badge to the main title in LoginPage.jsx for a more modern look and clear identification.

8. Resume Upload Feature:
   - Implemented a "Resume Upload" capability for both disbox and disbox-web.
   - Now, if an upload is interrupted and you try to upload the same file again (same name and size), the app will resume from where it left off.
   - This is achieved by generating a deterministic file ID (based on filename and size) and tracking successfully uploaded chunk message IDs in localStorage.
   - Once a file is fully uploaded and saved to the drive metadata, its resume data is automatically cleared.

9. Tab Close Warning:
   - Added a "beforeunload" event listener in AppContext.jsx to warn users when they attempt to close the browser tab or the app window while a file transfer is in progress.
   - This prevents accidental data loss from interrupted uploads or downloads.
   - Applied to both disbox and disbox-web.

10. Disbox Mobile Alignments:
    - Updated authentication flow: Guest access now calls the server-side verify webhook API to match the web version's security logic.
    - Implemented Resume Upload: Mobile now supports resuming interrupted uploads by using deterministic file IDs and tracking chunk progress in SharedPreferences.
    - Added getFileSize utility to support the resume logic.
    - Refactored upload use case and repository to support the new resume capability.

11. New Releases:
    - Desktop: Version 4.9.0 released successfully.
    - Mobile: Version 4.1.0 (build 11) released successfully.

12. Release Policy Update:
    - Both Desktop and Mobile repositories are now configured to only keep the SINGLE most recent release tagged as 'latest'.
    - Old releases and tags have been manually purged.
    - Workflows (.github/workflows) have been updated to automatically delete the previous 'latest' release before creating a new one on every successful build.
